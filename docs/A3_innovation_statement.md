# A3 项目创新说明书

**项目名称**：大模智学  
**赛题名称**：A3-基于大模型的个性化资源生成与学习多智能体系统开发  
**版本**：v2.0  
**最后更新**：2026-06-15

---

## 1. 创新定位

"大模智学"不是简单的"对话窗口+大模型API"缝合产品，而是将**学习画像**作为中心数据资产，让**7类专业智能体分工协作**，贯穿整个学习生命周期的系统化解决方案。项目的创新价值在于：将大模型从"对话能力"提升到"教学系统能力"，并且在RAG可信性、多Agent协同、评测闭环三个维度实现了可验证的工程落地。

---

## 2. 六大创新点深度分析

### 2.1 创新一：对话式画像驱动全链路个性化

**问题**：传统学习系统要求学生填写冗长表单或依靠静态标签来描述用户特征，信息采集率低、更新不及时、个性化效果差。

**创新方案**：
- 通过自然语言多轮对话自动采集信息（"我是大二计科学生，数据结构学得不好"→提取专业、年级、薄弱点）
- 10维画像体系：知识基础、认知风格(4型)、易错点、学习速度(3级)、兴趣、目标、偏好、薄弱、习惯、时间
- **关键词匹配+LLM理解**双重机制：中文关键词集推断认知风格/学习速度，LLM从对话中提取语义级信息
- 置信度递增设计：`confidence = len(extracted_info) / 10`，随信息增加逐步提升
- 画像作为**所有后续智能体的输入依据**：ResourceGenerator注入画像到prompt，Planner根据学习速度调整步长因子

**对比优势**：
| 维度 | 传统方案 | 本系统 |
|------|---------|--------|
| 信息采集 | 填表/标签 | 自然对话 |
| 维度数量 | 2-4个 | 10个 |
| 更新机制 | 手动 | 自动动态更新 |
| 下游利用 | 简单标签筛选 | 注入所有Agent prompt |

---

### 2.2 创新二：多智能体专业分工协同架构

**问题**：大多数LLM学习产品是"一个System Prompt + 一个对话窗口"的单体架构，难以在答辩中解释为什么能形成闭环，也难以扩展新功能。

**创新方案**：
- 7类Agent按角色清晰分工（非功能划分，而是按教学角色划分）
- **协调器单例模式**统一调度：`get_coordinator()` 全局单例，管理会话状态生命周期
- **并行资源生成**：`ThreadPoolExecutor(max_workers=4)` 同时生成多种资源
- **执行日志追溯**：每次Agent调用记录 `{timestamp, agent, action, params}`，可用于调试和演示
- 清晰的依赖关系：Profile → Resource/Planner → Tutor → Evaluator → (反馈回Profile)

**架构创新点**：
```
不是: 用户 → 一个大模型 → 回答
而是: 用户 → Coordinator → 画像Agent(10维)
                         → 资源Agent(6种Prompt) [并行]
                         → 规划Agent(算法驱动)
                         → 辅导Agent(6种答案类型)
                         → 评估Agent(8维加权)
                         → 知识库Agent(RAG全链路)
```

**对比优势**：
| 维度 | 单体LLM产品 | 本系统 |
|------|-----------|--------|
| 架构清晰度 | 一个Prompt解决一切 | 7个Agent各司其职 |
| 功能扩展 | 修改Prompt全文 | 新增Agent类 |
| 并行能力 | 串行 | ThreadPool并行 |
| 答辩可解释性 | "模型自己决定的" | "画像→资源→规划→辅导→评估" |

---

### 2.3 创新三：RAG可信问答的六层工程保障

**问题**：大多数参赛作品在文档中声称有RAG能力，但实际只有文件上传功能，回答仍是LLM自由生成，缺乏真正的证据检索和引用追溯。

**创新方案——六层工程保障**：

| 层级 | 机制 | 实现 |
|:---:|------|------|
| 第1层 | **8格式文档解析** | TXT/MD/CSV/JSON (编码检测) + DOCX (OpenXML) + PDF (PyPDF2) + PPTX (逐slide) + PPT (二进制提取) |
| 第2层 | **智能分块** | chunk_size=800, overlap=120, 优先在句号/换行处切割 |
| 第3层 | **双后端检索** | TF-IDF (char_wb ngram, 零依赖) + ChromaDB (可选向量检索, 自动切换) |
| 第4层 | **LLM证据生成** | System Prompt "仅根据材料回答，禁止编造", temperature=0.2, max_tokens=900 |
| 第5层 | **引用追溯** | 回答自动附加 `[来源1] 文档名 (文件名)` 引用列表 |
| 第6层 | **多重兜底** | Mock回答检测→模板拼接→检索片段直接展示→"资料不足"提示 |

**创新亮点**："资料不足"不是bug，是feature——未命中材料时明确告知而非强行回答，这恰恰是教学场景的核心需求。

### 2.4 创新四：评测结果回流学习规划的闭环设计

**问题**：传统学习系统把"测试"和"学习"割裂为两个独立功能，测试分数仅用于展示，不会影响后续学习安排。

**创新方案**：
```
能力测试 → quiz_results
         → EvaluatorAgent.evaluate() → 8维指标 → AssessmentReport
            ├── 雷达图可视化 (ECharts)
            ├── recommendations → 回流到画像 (更新 knowledge_base)
            └── adjust_path()
                ├── score < 0.6 → difficulty='easy', duration×1.2, +额外练习
                └── score > 0.9 → duration×0.8, +进阶挑战
```

**闭环价值**：
- 测试不是为了打分，而是为了**诊断**（8维细分远比"总分80"有用）
- 诊断结果**自动驱动**后续学习路径调整（非手动）
- 形成"学习→测试→诊断→调整→再学习"的持续优化循环

---

### 2.5 创新五：课程教学场景的深度适配

**问题**：大多数AI学习产品是泛化工具，缺乏与具体课程教学的结合，教师无法参与。

**创新方案**：
- **教师角色**：独立的教师分析工作台 + 课程材料上传入口 + 学生表现查看
- **教师材料优先**：当知识与教师上传讲义存在差异时，优先以教师材料为准
- **课程样例包**：提供"计算机组成原理"6个真实课程文件（讲义/题库/术语表/复习提纲/PPT），可直接用于演示
- **双专业方向**：能力测试同时覆盖普通高校专业方向和民航特色方向

## 3. 对比传统方案的差异化优势

| 维度 | 传统平台 | 大模智学 |
|------|---------|---------|

## 4. 当前阶段的现实优势

1. **本地可运行**：无需外部服务即可启动完整系统（LLM降级到模板/本地Ollama）
2. **演示链路完整**：从登录→画像→测试→报告→计划→RAG→资源生成，全程可演示
3. **知识库可信验证**：6个真实课程文件 + 上传/索引/检索/问答 全链路可核验
4. **工程化资料齐全**：Docker+一键启动+8类文档+证据包+PPT
5. **答辩展示性强**：甘特图/雷达图/思维导图/赛题对照页 视觉效果丰富

---

## 5. 后续可强化的创新方向

1. **向量数据库升级**：从TF-IDF升级到ChromaDB/Milvus，提升语义检索精度（已有切换接口）
2. **长期成长可视化**：画像前后变化对比、学习曲线追踪
3. **自动定期评估**：定时触发能力测试，形成持续诊断机制
4. **Agent数量扩展**：PPT生成Agent、视频渲染Agent、知识卡片Agent专业化拆分
5. **协作学习**：学习小组、讨论区、同伴互评
6. **多模态输入**：支持手写/语音/拍照提问

---

## 6. 结论

"大模智学"的核心创新不是"用了大模型"，而是**用系统化的多Agent架构让大模型从对话工具进化为教学系统**。画像驱动个性化、多Agent专业分工、RAG六层保障可信、评测回流形成闭环——这四个创新维度构成了一个完整的学习支持生态，使系统不只是"回答问题的工具"，而是能够持续理解学生、适应学生、引导学生成长的教学支持平台。这种"画像驱动、多智能体协同、知识库兜底、测评回流优化"的设计，更贴近高校个性化学习的真实场景，也更符合软件杯A3赛题对系统性与落地性的核心要求。
