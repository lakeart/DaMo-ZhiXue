## 1. 手册用途

## 2. 环境与启动

### 2.1 系统要求

| 项目 | 最低要求 |
|------|---------|
| 操作系统 | Windows 10/11 或 Linux/macOS |
| Python | ≥ 3.8 |
| 内存 | ≥ 4GB |
| 磁盘 | ≥ 500MB 空闲空间 |
| 网络 | 可选（LLM API需要联网） |

### 2.2 本地启动

**方式一：手动启动**
```bash
cd 1
pip install -r requirements.txt
python run.py
```

**方式二：一键启动（Windows）**
```
双击 start.bat
```

### 2.3 访问地址

```
http://127.0.0.1:5000
```

## 4. 核心功能详细操作指南

### 4.1 能力测试中心

**路径**：首页 → 能力测试

**操作步骤**：
1. 以学生账号登录（`student / student123`）
2. 进入"能力测试中心"页面
3. 选择专业方向：
   - **普通高校专业方向**：计算机科学基础题库
   - **民航特色方向**：民航专业特色题目
4. 按顺序逐题作答（选择题为主）
5. 全部完成后点击"提交"
6. 查看测试结果：
   - **总分**：综合得分（百分制）
   - **难度分布**：easy/medium/hard 三级分类统计
   - **逐题解析**：每题显示用户答案、正确答案、详细解析
   - **建议**：基于答题表现的个性化学习建议

**页面元素**：题目卡片、选项按钮、进度条、计时器、提交按钮

**后续操作**：可从结果页直接跳转到"学习报告"或"学习计划"页面

---

### 4.2 学习报告

**路径**：能力测试结果页 → 学习报告，或导航栏 → 学习报告

**操作步骤**：
1. 完成能力测试后点击"查看学习报告"
2. 查看**总体评估**：综合得分 + 等级评定（优秀/良好/一般/需改进）
3. 查看**能力雷达图**：8维度直观展示
   - 知识掌握度、练习准确率、概念理解度、问题解决能力
   - 学习效率、时间管理、学习持续性、学习投入度
4. 查看**优势领域**和**需要加强**的指标
5. 查看**学习趋势**：知识点进步/退步情况
6. 查看**改进建议**：个性化学习指导（最多5条）

**可视化说明**：雷达图使用ECharts渲染，8轴分别对应8个评估维度，蓝色区域越大表示能力越全面。支持悬停查看具体分数。

**后续操作**：点击"基于评估调整学习计划"将评估结果回流到学习路径中

---

### 4.3 学习计划

**路径**：导航栏 → 学习计划，或学习报告页 → 进入学习计划

**操作步骤**：
1. 首次进入：系统基于画像自动生成个性化学习路径
2. 支持三种视图切换：

   **甘特图视图**：
   - 时间线横向展示所有学习步骤
   - 5个里程碑标记（入门成功→基础掌握→能力提升→深入理解→融会贯通）
   - 颜色编码难度（绿=简单/蓝=中等/红=困难）

   **日历视图**：
   - 7天网格布局
   - 每天显示分配到的学习任务
   - 每项任务含标题、知识点、预估时长

   **列表视图**：
   - 步骤编号顺序列表
   - 每项显示：标题、描述、知识点、活动、难度标签、时长

3. **周计划**：在页面底部查看7天日程分配表
4. **AI洞察面板**：显示学习策略、风险提醒、资源推荐

**动态调整**：完成能力测试后重新进入学习计划，路径会根据评估结果自动调整（难度/时长/活动变化）

---

### 4.4 智能问答（多智能体模式）

**路径**：导航栏 → 智能问答

这是系统最核心的页面，集成了**普通问答**和**多智能体协同**两种模式。

**4.4.1 普通问答模式**
1. 在输入框输入问题，如"快速排序的时间复杂度是多少？"
2. 按回车发送
3. 系统返回结构化回答（问题理解→解答→知识延伸→思考题）
4. 支持**流式回答**：答案逐字显示
5. 支持**追问纠错**：在回答下继续提问，系统保持上下文
6. 支持**多模态检测**：提问含代码自动切换代码解答格式

**4.4.2 多智能体模式**
1. 切换到"多智能体模式"标签
2. **第一步：构建学习画像**
   - 输入："我是大二计科学生，数据结构学得不好，准备考研408"
   - 系统对话回复，右侧面板显示10维画像动态更新
3. **第二步：资源生成**
   - 选择知识点（如"数据结构-链表"）
   - 选择资源类型（讲义/思维导图/练习题/拓展阅读/视频脚本/代码实操）
   - 点击生成，可同时选择多种
   - 查看生成的资源卡片（标题、预览、类型图标、难度标签）
4. **第三步：创建学习计划**
   - 输入学习目标
   - 设置每周学习时间
   - 生成个性化学习路径
5. **第四步：智能辅导**
   - 在问答区域继续提问
   - 系统基于画像提供个性化回答

---

### 4.5 知识库管理与RAG问答

**路径**：智能问答页 → 知识库管理区域

**4.5.1 上传课程资料**
1. 在知识库管理面板点击"上传文档"
2. 选择课程资料文件（支持8种格式）
3. 支持拖拽和批量上传
4. 系统自动完成：
   - 文件解析（TXT/MD/CSV/JSON编码读取，DOCX/PDF/PPTX XML解析）
   - 文本分块（chunk_size=800字符，overlap=120字符）
   - 关键词提取（top 12）
   - 检索引擎建立索引
   - SHA256去重检测

**支持文件格式**：
| 格式 | 扩展名 | 说明 |
|------|--------|------|
| 纯文本 | `.txt` | 自动检测编码 |
| Markdown | `.md` | 保留格式 |
| CSV | `.csv` | 表格数据 |
| JSON | `.json` | 结构化数据 |
| Word | `.docx` | OpenXML解析 |
| PDF | `.pdf` | 文本层提取 |
| PPTX | `.pptx` | 逐slide提取 |
| PPT | `.ppt` | 兼容提取 |

**4.5.2 知识库问答**
1. 确保已上传课程资料
2. 在问答输入框提问，如"什么是冯诺依曼架构？"
3. 系统自动：
   - 在知识库中检索相关分块（TF-IDF/ChromaDB）
   - 将检索结果作为上下文发送给LLM
   - LLM基于证据生成回答
   - 回答末尾附加`[来源N]`引用标记
4. 输出包含：**回答正文** + **引用列表**（文档名、分块位置、相似度分数） + **置信度**
5. 如果知识库中没有相关资料，系统返回"资料不足"提示并建议上传材料

**4.5.3 管理已上传文档**
- 查看文档列表（标题/文件名/类型/大小/分块数/上传时间）
- 查看检索引擎状态（TF-IDF/ChromaDB）
- 查看支持的格式列表

### 4.6 学习画像构建

**路径**：导航栏 → 学习画像，或智能问答多智能体模式 → 画像构建

**操作步骤**：
1. 在对话框输入你的学习情况
2. 系统通过自然对话收集信息：
   - "我是大二计算机科学与技术专业的学生"
   - "数据结构学得不太好，特别是链表和树"
   - "平时喜欢通过看视频学习，不太喜欢读教材"
   - "每天大概有2-3小时可以用于学习"
3. 右侧面板实时显示以下维度的更新：
   - 知识基础：各知识点掌握度
   - 认知风格：visual（视觉型）
   - 学习速度：medium（中等）
   - 兴趣方向：算法、编程
   - 学习目标：考研408
   - 薄弱知识点：链表、树
   - 偏好/强项/习惯/时间...
4. 系统会智能追问缺失的维度信息
5. 画像置信度随信息增加而提升

## 8. 常见问题

### Q1: 页面打不开怎么办？
1. 检查 `python run.py` 或 `start.bat` 是否正常启动
2. 确认访问地址为 `http://127.0.0.1:5000`（不是localhost）
3. 端口被占用：关闭旧进程后重启
4. 检查控制台是否有错误日志

### Q2: 文档上传失败？
1. 检查文件扩展名是否在支持列表中
2. 文件内容过少（< 20字符）会被拒绝
3. 扫描版PDF（纯图片）无法提取文本，建议先OCR
4. 损坏的Office文件无法解析

### Q3: 知识库问答显示"资料不足"？
1. 说明当前上传的材料未命中足够的相关内容
2. 建议补充课程讲义、实验文档、课件和题库
3. 优先上传**教师材料**（系统优先使用教师来源）
4. 增加上传文件数量可以提升命中率

### Q4: LLM调用失败/返回mock回答？
1. 系统在无真实API Key时使用模板化降级回答
2. 如需真实LLM效果：
   - 配置环境变量 `OPENAI_API_KEY` + `OPENAI_BASE_URL`（OpenAI兼容API）
   - 或启动本地Ollama服务（`ollama serve`，默认`localhost:11434`）
3. Mock回答会被系统检测并替换为检索材料拼接

### Q5: 演示时如何降低风险？
1. 优先使用本地知识库链路（不依赖外部API）
2. 准备已生成好的资源截图作为备用
3. `start.bat` 和 Docker 都已验证可用
4. 课程样例包已完成真实验证
