## 2. 环境与启动

### 2.1 系统要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10/11 或 Linux/macOS | Windows 11 |
| Python | ≥ 3.8 | 3.11 (venv) |
| 内存 | ≥ 4GB | ≥ 8GB |
| 磁盘 | ≥ 500MB 空闲空间 | ≥ 2GB (含依赖) |
| 网络 | 可选（LLM API需要联网） | 联网(LLM真实效果更佳) |
| 浏览器 | Chrome 90+ / Edge 90+ | Chrome 120+ |

### 2.2 本地启动

**方式一：手动启动（推荐开发/调试）**
```bash
cd damo-zhixue
pip install -r requirements.txt
python run.py
```

**方式二：一键启动（Windows，推荐演示）**
```
双击 start.bat
```
脚本自动执行：检测Python → 创建venv(可选) → 安装依赖 → 初始化数据库 → 启动服务器

**方式三：Docker部署（推荐生产/演示）**
```bash
docker-compose up -d
# 访问 http://localhost:80
```

### 2.3 访问地址

```
http://127.0.0.1:5000
```

> 注意：使用 `127.0.0.1` 而非 `localhost`，避免某些浏览器DNS解析延迟。

### 2.4 演示账号

| 角色 | 用户名 | 密码 | 登录后跳转 | 典型用途 |
|------|--------|------|----------|---------|
| 管理员 | `admin` | `admin123` | 全局管理页面 | 知识库管理+RAG演示 |

启动成功后控制台显示：

## 4. 核心功能详细操作指南

### 4.1 能力测试中心

**路径**：首页 → 能力测试  
**适用角色**：学生  
**涉及Agent**：EvaluatorAgent

**操作步骤**：
1. 以学生账号登录（`student / student123`）
2. 进入"能力测试中心"页面
3. 选择专业方向：
   - **普通高校专业方向**：计算机科学基础题库（数据结构/计组/操作系统/网络）
   - **民航特色方向**：民航专业特色题目（航空通信/导航/监视）
4. 按顺序逐题作答（选择题为主，每页1题）
5. 全部完成后点击"提交"
6. 查看测试结果：
   - **总分**：综合得分（百分制）+等级评定
   - **难度分布**：easy/medium/hard 三级分类统计（含正确率）
   - **逐题解析**：每题显示用户答案、正确答案、详细解析
   - **知识点分析**：各知识点的正确率与薄弱点标记

**页面元素详解**：
- 题目卡片：居中显示，含题号/题型/题干/4个选项
- 进度条：顶部显示当前进度（N/总数）
- 计时器：每个方向独立计时（非强制，仅记录用时）
- 上一题/下一题：可回看和修改答案
- 提交按钮：最后确认提交前可预览所有答案

**演示技巧**：
- 快速作答2-3题即可展示功能（不必全部做完）
- 重点展示"双方向切换"和"逐题解析"两个亮点

**后续操作**：可从结果页直接跳转到"学习报告"或"学习计划"页面

### 4.2 学习报告

**路径**：能力测试结果页 → 学习报告，或导航栏 → 学习报告  
**适用角色**：学生  
**涉及Agent**：EvaluatorAgent

**操作步骤**：
1. 完成能力测试后点击"查看学习报告"
2. 查看**总体评估**：综合得分 + 等级评定（优秀≥85/良好≥70/一般≥55/需改进<55）
3. 查看**能力雷达图**：8维度直观展示
   - 知识掌握度(0.20)、练习准确率(0.18)、概念理解度(0.15)、问题解决能力(0.15)
   - 学习效率(0.10)、时间管理(0.08)、学习持续性(0.07)、学习投入度(0.07)
   - 括号内为各维度权重
4. 查看**优势领域**（score>0.75，绿色标记）
5. 查看**需要加强**（score<0.55，红色标记）
6. 查看**学习趋势**：与上次测试对比的进步/退步知识点
7. 查看**改进建议**：个性化学习指导（最多5条，按优先级排序）

**雷达图交互说明**：
- 8轴分别对应8个评估维度，使用ECharts渲染
- 蓝色填充区域表示各维度得分，面积越大=能力越全面
- 鼠标悬停任意轴可查看具体分数
- 支持tooltip显示维度名称+得分+权重
- 雷达图随窗口大小自适应缩放

**演示技巧**：
- 先展示总分→切换到雷达图（视觉亮点）
- 指出"8维评估"vs"传统总分"的差异
- 强调"改进建议"是基于薄弱维度自动生成的
- 点击"基于评估调整学习计划"展示闭环

### 4.3 学习计划

**路径**：导航栏 → 学习计划，或学习报告页 → 进入学习计划  
**适用角色**：学生  
**涉及Agent**：LearningPlannerAgent

**操作步骤**：
1. 首次进入：系统基于画像自动生成个性化学习路径
2. 支持三种视图切换：

   **甘特图视图**（推荐演示首选）：
   - 时间线横向展示所有学习步骤
   - 5个里程碑标记：入门成功→基础掌握→能力提升→深入理解→融会贯通
   - 颜色编码难度：绿(easy)/蓝(medium)/红(hard)
   - 鼠标悬停显示步骤详情（标题/知识点/时长）
   - 适用于答辩展示，视觉冲击力强

   **日历视图**：
   - 7天网格布局（周视图）
   - 每天显示分配到的学习任务
   - 每项任务含：标题、知识点标签、预估时长
   - 彩色标签区分任务类型（学习/练习/复习/测试）
   - 适用于日常学习执行

   **列表视图**：
   - 步骤编号顺序列表
   - 每项显示：标题、描述、知识点、活动类型、难度标签、预估时长
   - 支持展开/折叠详细描述
   - 适用于打印和详细查看

3. **周计划**：在页面底部查看7天日程分配表（每天任务数+总时长）
4. **AI洞察面板**（右侧/底部）：
   - 学习策略：基于画像的建议（如"你的学习速度较慢，建议每步多留20%时间"）
   - 风险提醒：薄弱知识点预警
   - 资源推荐：匹配当前步骤的资源卡片

**动态调整机制演示**：
- 先展示当前学习路径（记录某个step的difficulty和duration）
- 完成一次能力测试
- 重新进入学习计划
- 观察：薄弱知识点对应的步骤难度降低、时长增加、新增额外练习

**演示技巧**：
- 先展示甘特图（30秒，视觉冲击）
- 切换到日历视图（展示可用性）
- 指出不同颜色代表不同难度（证明个性化）
- 展示底部AI洞察面板（证明画像驱动）

### 4.4 智能问答（多智能体模式）

**路径**：导航栏 → 智能问答  
**适用角色**：学生  
**涉及Agent**：TutorAgent + ProfileBuilderAgent + ResourceGeneratorAgent

这是系统最核心的页面，集成了**普通问答**和**多智能体协同**两种模式。

**4.4.1 普通问答模式**

操作：
1. 在输入框输入问题，如"快速排序的时间复杂度是多少？"
2. 按Enter发送
3. 系统返回结构化回答：
   ```
   📝 问题理解: 您想了解快速排序算法的时间复杂度分析
   📖 解答: 快速排序的... [详细解答含公式]
   💡 知识延伸: 与其他排序算法的对比...
   ❓ 思考题: 如果选择第一个元素作为pivot...
   ```
4. 支持**流式回答**：答案逐字显示（SSE推送）
5. 支持**追问纠错**：在回答下继续提问，系统保持上下文
6. 支持**多模态检测**：提问含代码自动切换代码解答格式（带语法高亮）

**4.4.2 多智能体模式**（答辩核心展示）

切换到"多智能体模式"标签，按以下步骤操作：

**第一步：构建学习画像（45秒）**
- 输入："我是大二计科学生，数据结构学得不好，准备考研408"
- 系统对话回复，右侧面板显示10维画像动态更新
- 继续输入信息补充画像（如学习偏好、可用时间等）
- 观察：画像置信度从0逐渐升至>0.3
- **演示要点**：强调"无需填表，自然对话即可"

**第二步：资源生成（45秒）**
- 选择知识点（如"数据结构-链表"）
- 选择资源类型（可多选）：
  - 📄 课程讲义 — Markdown格式，800-1500字
  - 🧠 思维导图 — Mermaid格式，3-4层
  - ✏️ 练习题 — 5题型×3难度
  - 📚 拓展阅读 — 3-5个推荐资源
  - 🎬 视频脚本 — 4-6镜头分镜
  - 💻 代码实操 — Python完整代码+注释
- 点击生成，可同时选择多种
- 查看生成的资源卡片（标题、预览、类型图标、难度标签）
- **演示要点**：先选1-2种快速展示，再展示并行生成

**第三步：创建学习计划（30秒）**
- 输入学习目标（预填画像中的目标）
- 设置每周学习时间（预填画像中的可用时间）
- 点击生成个性化学习路径
- **演示要点**：路径自动适配画像中的学习速度和薄弱点

**第四步：智能辅导（30秒）**
- 在问答区域继续提问
- 系统基于画像提供个性化回答（风格/难度匹配）
- **演示要点**：比较同一问题"有画像"vs"无画像"回答的差异

### 4.5 知识库管理与RAG问答

**路径**：智能问答页 → 知识库管理区域  
**适用角色**：管理员/教师  
**涉及Agent**：KnowledgeBaseAgent + TutorAgent

**4.5.1 上传课程资料**

操作步骤：
1. 在知识库管理面板点击"上传文档"
2. 选择课程资料文件（支持8种格式）
3. 支持拖拽和批量上传

系统自动完成后台处理流水线：
```
文件上传
  → 格式检测 (扩展名+文件头验证)
  → 文本提取
     ├─ TXT/MD/CSV/JSON: 编码检测(chardet) + 直接读取
     ├─ DOCX: zipfile解压 → ElementTree解析word/document.xml
     ├─ PDF: PyPDF2 PdfReader逐页提取
     ├─ PPTX: zipfile解压 → 逐slide提取<a:t>元素
     └─ PPT: 二进制正则提取可读字符串
  → SHA256哈希去重 (与已索引文件比对)
  → 智能分块 (chunk_size=800, overlap=120, 句号优先)
  → 关键词提取 (TF-IDF top-12)
  → 检索引擎建立索引 (TF-IDF向量 or ChromaDB嵌入)
  → 返回索引结果 (文档ID, 分块数, 关键词)
```

**支持文件格式**：

| 格式 | 扩展名 | 解析方式 | 注意事项 |
|------|--------|---------|---------|
| 纯文本 | `.txt` | 编码检测+直接读取 | 自动检测UTF-8/GBK等编码 |
| Markdown | `.md` | 同TXT+去标记 | 保留标题层级结构信息 |
| CSV | `.csv` | csv.reader | 自动检测分隔符(`,`/`\t`/`;`) |
| JSON | `.json` | json.loads递归提取 | 嵌套对象展平为文本 |
| Word文档 | `.docx` | zipfile+ElementTree | 支持段落+表格文本 |
| PDF文档 | `.pdf` | PyPDF2提取 | 仅文本型，扫描版需OCR |
| PPTX演示 | `.pptx` | zipfile+ElementTree | 逐slide提取文本 |
| PPT演示 | `.ppt` | 二进制正则提取 | 兼容提取，非100% |

**4.5.2 知识库问答**

操作步骤：
1. 确保已上传课程资料（推荐上传3-4个样例文件）
2. 在问答输入框提问，如"什么是冯诺依曼架构？"
3. 系统后台处理：
   ```
   用户提问 → 查询预处理(关键词提取)
     → 知识库检索(TF-IDF/ChromaDB, top-k=5)
     → 相似度过滤(阈值0.3)
     → 构建LLM上下文(检索材料拼接)
     → LLM证据生成(temperature=0.2)
     → 后处理:
        ├─ Mock检测(识别"作为AI助手"等模板话术)
        ├─ 幻觉检测(回答与材料交叉验证)
        └─ 引用附加(标注[来源N])
     → 返回用户
   ```
4. 输出结构：
   - **回答正文**：基于材料生成的内容
   - **📚 参考来源**：`[来源N] 文档名 (分块信息) — 相似度: 0.XX`
   - **置信度**：回答的可信程度（基于检索命中率）
5. 如果知识库中没有相关资料，系统返回：
   ```
   根据现有资料无法确定您的问题。
   💡 建议: 请上传相关的课程资料（讲义、教材、课件等），
   我将基于您提供的材料为您解答。
   ```

**4.5.3 管理已上传文档**

管理面板功能：
- 查看文档列表（标题/文件名/类型/大小/分块数/上传时间/来源）
- 查看检索引擎状态（TF-IDF/ChromaDB）
- 查看支持的格式列表和上传建议
- 查看知识库统计（总文档数/总分块数/总字符数）

**推荐演示问题序列**：
| 序号 | 问题 | 预期效果 | 依赖材料 |
|:--:|------|---------|---------|
| 1 | "什么是冯诺依曼架构？" | 带引用的完整回答 | 讲义/复习提纲 |
| 2 | "Cache的主要作用是什么？" | 多材料交叉引用 | 讲义+复习提纲 |
| 3 | "指令流水线的五个阶段" | 精确术语匹配 | 术语表+讲义 |
| 4 | "量子纠缠是什么？" | 资料不足拦截 | 无(故意测试) |

### 4.6 学习画像构建

**路径**：导航栏 → 学习画像  
**适用角色**：学生

## 8. 常见问题与故障排查

### Q1: 页面打不开怎么办？

**排查步骤**：
1. 检查 `python run.py` 或 `start.bat` 是否正常启动（控制台应有"Running on"信息）
2. 确认访问地址为 `http://127.0.0.1:5000`（不是`localhost`）
3. 端口被占用：
   ```bash
   # 查看端口占用
   netstat -ano | findstr :5000
   # 或修改run.py中的端口号
   ```
4. 检查控制台是否有错误日志（红色traceback信息）
5. Windows防火墙弹窗 → 点击"允许访问"

### Q2: 文档上传失败？

**排查步骤**：
1. 检查文件扩展名是否在支持列表中（`.txt/.md/.csv/.json/.docx/.pdf/.pptx/.ppt`）
2. 文件内容过少（<20字符）会被拒绝 → 用记事本打开检查
3. 扫描版PDF（纯图片）无法提取文本 → 建议OCR后上传
4. 损坏的Office文件无法解析 → 尝试用Office软件打开确认完整
5. 大文件（>10MB）可能超时 → 拆分或压缩

### Q3: 知识库问答显示"资料不足"？

**说明**：这是系统设计行为，不是bug。

**解决**：
1. 当前上传的材料未命中足够的相关内容
2. 建议补充课程讲义、实验文档、课件和题库
3. 优先上传**教师材料**（系统优先使用教师来源）
4. 增加上传文件数量可以提升命中率
5. 如果确定材料中有相关内容但仍显示"资料不足"，检查检索后端设置

### Q4: LLM调用失败/返回Mock回答？

**说明**：系统在无真实API Key时使用模板化降级回答。

**如需真实LLM效果**：
```bash
# 选项A: 配置OpenAI兼容API
$env:OPENAI_API_KEY="sk-..."
$env:OPENAI_BASE_URL="https://api.openai.com/v1"

# 选项B: 启动本地Ollama
ollama pull qwen2.5:7b
ollama serve
# 系统自动探测 localhost:11434
```

**识别Mock回答**：
- 回答中包含"作为AI助手"等模板话术
- 回答末尾标注"(模板生成)"

**系统保护**：Mock回答会被自动检测并替换为检索材料直接拼接展示，确保不输出虚假信息。

### Q5: 演示时如何降低风险？

**风险规避策略**：

| 风险 | 规避方案 |
|------|---------|
| 网络中断 | 优先使用本地知识库链路（不依赖外部API） |
| LLM API限流 | 准备已生成好的资源截图作为备用 |
| 系统崩溃 | `start.bat` 和 Docker 均已验证，可快速重启 |
| 演示环境差异 | 提前在演示电脑上运行一遍`start.bat` |
| 时间不足 | 按§7"时间紧张版"脚本执行 |

### Q6: 密码错误或账号被锁定？

1. 重新运行 `python set_db.py` 重置演示账号
2. 确认用户名密码输入正确（大小写敏感）
3. 管理员账号不存在锁定机制

### Q7: 雷达图/甘特图/Mermaid图不显示？

1. 确认浏览器支持JavaScript
2. 确认CDN资源可加载（检查浏览器F12 Console）
3. ECharts/Mermaid通过CDN加载，需要联网
4. 如果完全离线：可将CDN资源下载到static目录做本地引用

### Q8: Docker部署后无法访问？

1. 确认容器正在运行：`docker ps`
2. 检查端口映射：`docker-compose.yml`中`5000:80`映射
3. 访问地址：`http://localhost:80`（Docker映射到80端口）
4. 检查容器日志：`docker-compose logs`
